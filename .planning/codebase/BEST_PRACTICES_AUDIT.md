### 2. Test Organization ⚠️

**Current State:**
- Tests are embedded within source files (unit tests in `cli.rs`, `notion.rs`, etc.)
- No separate `tests/` directory for integration tests
- 34 tests total, all passing

**Follows Best Practices:** ⚠️ Partially

**Issues Found:**

| Issue | Severity | Recommendation |
|-------|----------|----------------|
| No separate `tests/` directory for integration tests | Medium | Create `tests/integration.rs` for end-to-end tests that require network access to Notion API |
| No example tests in `tests/` directory | Low | Add integration test examples for common use cases |

**Recommendations:**
```bash
# Recommended structure:
tests/
├── integration.rs          # Integration tests (API calls)
└── ui/                     # UI tests if applicable
```

**Rationale:** Unit tests in source files are appropriate for pure functions. Integration tests should be separate to:
- Allow longer-running tests
- Enable mocking strategies
- Separate test compilation from library compilation

### 10. Binary Naming ⚠️

**Current State:**
- Package name: `notion-sql` (with hyphen)
- Binary name: `notion-sql`
- Library crate: `notion_sql` (with underscore)

**Follows Best Practices:** ⚠️ Partially

**Issue:**
- Package name uses hyphen (`notion-sql`)
- Library crate uses underscore (`notion_sql`)

**Rust Convention:**
According to [RFC 2517](https://rust-lang.github.io/rfcs/2517-crate-name-reexported-as-crate-name.html), the crate name should match the package name when possible.

**Recommendations:**

| Option | Package Name | Crate Name | Pros | Cons |
|--------|-------------|------------|------|------|
| **A (Current)** | `notion-sql` | `notion_sql` | Follows Cargo convention for package names | Inconsistent naming |
| **B (Recommended)** | `notion_sql` | `notion_sql` | Consistent naming, follows RFC 2517 | Changes package name |
| **C (Acceptable)** | Keep as-is | Keep as-is | No breaking changes for users | Inconsistent naming |

**If choosing Option B:**
1. Update `Cargo.toml`: `name = "notion_sql"`
2. Update README.md installation instructions
3. Update Homebrew tap references
4. Update documentation

**Current Impact:** Low - Users interact with binary name `notion-sql`, so the hyphenated package name is acceptable for CLI tools.
